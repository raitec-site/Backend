# Manual Git — Projeto RAITec

# 1. Objetivo
Este documento tem como objetivo orientar os desenvolvedores sobre:

- Configuração inicial do projeto
- Clonagem do repositório
- Instalação das dependências
- Configuração do Firebase
- Uso correto do Git
- Fluxo de branches
- Commits
- Atualizações do projeto
- Organização da equipe



# 2. Regras importantes

- Nunca subir `.env`
- Nunca subir `serviceAccountKey.json`
- Nunca trabalhar diretamente na `main`
- Sempre atualizar a `development`
- Cada funcionalidade deve possuir sua própria branch
- Não alterar código de outras funcionalidades sem alinhamento



# 3. Resultado esperado

Ao final, o desenvolvedor deve conseguir:

- Clonar o projeto
- Configurar Firebase
- Rodar o sistema
- Entender a estrutura de branches
- Trabalhar corretamente na branch da funcionalidade
- Fazer commits padronizados
- Atualizar o projeto
- Trabalhar em equipe sem conflitos
